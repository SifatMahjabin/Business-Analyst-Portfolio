# ⚡ Sales Data Modeling with Excel Power Pivot

## Project Overview

This project demonstrates the use of Microsoft Excel Power Pivot to build a relational data model for sales analysis. Instead of relying on a single flat dataset, multiple related tables were connected using primary and foreign keys to create an efficient data model for reporting and business analysis.

---

## Business Objective

The objective of this project is to organize sales data into a structured relational model that supports faster analysis, improved reporting, and scalable business intelligence.

---

## Dataset

The project uses three related tables:

### Customers
- Customer ID
- Customer Name
- Phone
- Address
- City
- State
- Country

### Orders
- Order ID
- Customer ID
- Product
- Region
- Units Sold
- Order Date

### Cookie Types
- Cookie Type
- Revenue per Cookie
- Cost per Cookie

---

## Data Model

Relationships were created between tables using Power Pivot's Diagram View.

- Customers → Orders (One-to-Many)
- Cookie Types → Orders (One-to-Many)

This relational model enables accurate aggregation and eliminates duplicate data.

---

## Features Implemented

- Imported multiple tables into the Data Model
- Created table relationships
- Built a relational database model
- Used Diagram View to visualize relationships
- Managed data in Power Pivot Data View
- Prepared data for Pivot Tables and Dashboard reporting
